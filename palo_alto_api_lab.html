Module — Firewall Automation

Overview & Objectives

Automate a Palo Alto firewall in EVE-NG using VS Code and the PAN-OS REST API. Your local PC and the firewall share the same network — no jump host needed.

ℹ This module uses the PAN-OS XML API (port 443). The firewall's management interface must be reachable from your PC. All requests are HTTPS.
Lab Environment
🔥

Palo Alto Firewall

Running in EVE-NG. Management IP reachable from your local PC on the same subnet.

☁️

Cloud Management

Connected to Panorama or Strata Cloud Manager via management port.

💻

VS Code + Python

Your local machine. Python 3.x with the requests library installed.

🔗

Same Network

PC ↔ Firewall mgmt on same L3 network. Direct API calls — no proxy required.

What You Will Build

Lab Coverage

  • Lab 1 — Generate and store an API key from the firewall
  • Lab 2 — Retrieve all interface names, IPs, and link states
  • Lab 3 — Fetch and parse security policy rules
  • Lab 4 — Create a new address object (host / subnet)
  • Lab 5 — Commit candidate configuration to running config
  • Lab 6 — Pull system-level information (hostname, model, uptime)

api_client.py
import requests, xmltodict, urllib3
from config import BASE_URL, API_KEY, VERIFY_SSL

urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

def api_get(params):
    params["key"] = API_KEY
    r = requests.get(BASE_URL, params=params, verify=VERIFY_SSL)
    r.raise_for_status()
    return xmltodict.parse(r.text)

def api_post(params, data=None):
    params["key"] = API_KEY
    r = requests.post(BASE_URL, params=params, data=data, verify=VERIFY_SSL)
    r.raise_for_status()
    return xmltodict.parse(r.text)

Lab 1

Generate API Key

GETtype=keygen

Authenticate to the PAN-OS XML API using username and password to retrieve a long-lived API key. Store it in config.py for all subsequent labs.

API URL
https://{FIREWALL_IP}/api/?type=keygen&user=admin&password={PASSWORD}
Script
labs/lab1_get_apikey.py
import requests, xmltodict, urllib3
from config import FIREWALL_IP, USERNAME, PASSWORD, VERIFY_SSL

urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

def get_api_key():
    url    = f"https://{FIREWALL_IP}/api/"
    params = {
        "type"    : "keygen",
        "user"    : USERNAME,
        "password": PASSWORD,
    }
    response = requests.get(url, params=params, verify=VERIFY_SSL)
    data     = xmltodict.parse(response.text)

    if data["response"]["@status"] == "success":
        api_key = data["response"]["result"]["key"]
        print(f"[+] Key: {api_key[:20]}...")
        print("[!] Paste into config.py -> API_KEY")
        return api_key
    else:
        print("[-] Failed:", data)

if __name__ == "__main__":
    get_api_key()
Expected Response
<response status="success"> <result> <key>LUFRPT14MW5xOEo1R09KVlBZNnpnemh0VHRBOWl6TGM9...</key> </result> </response>
Lab 2

Get Interface Details

GETtype=op

Use an operational command to retrieve all interfaces, their IP addresses, MTU, and link state.

API URL
https://{FW_IP}/api/?type=op&cmd=<show><interface>all</interface></show>&key={API_KEY}
ℹ Building the cmd: CLI is show interface all. Wrap each word in XML tags: <show><interface>all</interface></show>. The requests library URL-encodes this automatically.
Script
labs/lab2_interfaces.py
from api_client import api_get

def get_interfaces():
    params = {
        "type": "op",
        "cmd" : "<show><interface>all</interface></show>",
    }
    data       = api_get(params)
    hw_entries = data["response"]["result"]["hw"]["entry"]
    if_entries = data["response"]["result"]["ifnet"]["entry"]

    if isinstance(hw_entries, dict): hw_entries = [hw_entries]
    if isinstance(if_entries, dict): if_entries = [if_entries]

    print(f"\n{'Interface':<15}{'IP':<20}{'State':<10}{'Speed'}")
    print("-" * 55)
    for hw in hw_entries:
        name  = hw.get("@name", "N/A")
        state = hw.get("state", "N/A")
        speed = hw.get("speed", "N/A")
        ip    = "N/A"
        for iface in if_entries:
            if iface.get("@name") == name:
                ip = iface.get("ip", "N/A"); break
        print(f"{name:<15}{ip:<20}{state:<10}{speed}")

if __name__ == "__main__":
    get_interfaces()

Lab 3

Get Security Policies

GETtype=config & action=get

Read the security rulebase from the candidate configuration using an XPath query.

API URL
https://{FW_IP}/api/?type=config&action=get&xpath=/config/devices/entry[@name='localhost.localdomain']/vsys/entry[@name='vsys1']/rulebase/security/rules&key={API_KEY}
ℹ XPath tip: Use action=get to read candidate config and action=show for running config. For automation, get is preferred.
Script
labs/lab3_security_policy.py
from api_client import api_get

SECURITY_XPATH = (
    "/config/devices/entry[@name='localhost.localdomain']"
    "/vsys/entry[@name='vsys1']/rulebase/security/rules"
)

def get_security_policies():
    params = {
        "type"  : "config",
        "action": "get",
        "xpath" : SECURITY_XPATH,
    }
    data  = api_get(params)
    rules = data["response"]["result"]["rules"]["entry"]
    if isinstance(rules, dict): rules = [rules]

    print(f"\n[+] Found {len(rules)} rule(s)\n")

    for i, rule in enumerate(rules, 1):
        print(f"Rule #{i}: {rule['@name']}")
        print(f"  Src Zone: {rule['from']['member']}")
        print(f"  Dst Zone: {rule['to']['member']}")
        print(f"  App    : {rule['application']['member']}")
        print(f"  Action : {rule.get('action','N/A').upper()}")
        print("-" * 45)

if __name__ == "__main__":
    get_security_policies()
Lab 4

Create Address Object

POSTtype=config & action=set

Push a new address object into the candidate configuration. Changes go live only after a commit (Lab 5).

API URL
POST https://{FW_IP}/api/?type=config&action=set&xpath=/config/devices/entry/vsys/entry[@name='vsys1']/address&element=<entry name="WEB-SERVER-01">...</entry>&key={API_KEY}
⚠ The element XML is a POST parameter, not the request body. Use data={} in requests, not json={}.
Script
labs/lab4_address_object.py
import requests, xmltodict, urllib3
from config import API_KEY, BASE_URL, VERIFY_SSL

urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

ADDR_XPATH = (
    "/config/devices/entry[@name='localhost.localdomain']"
    "/vsys/entry[@name='vsys1']/address"
)

def create_address_object(name, ip, description=""):
    print(f"[*] Creating: {name} ({ip})")
    element = f"""<entry name="{name}">
    <ip-netmask>{ip}</ip-netmask>
    <description>{description}</description>
</entry>"""

    params = {
        "type"   : "config", "action": "set",
        "xpath"  : ADDR_XPATH,
        "element": element,   "key"   : API_KEY,
    }
    resp = requests.post(BASE_URL, params=params, verify=VERIFY_SSL)
    data = xmltodict.parse(resp.text)

    if data["response"]["@status"] == "success":
        print(f"[+] '{name}' created! Run lab5_commit.py to activate.")
    else:
        print("[-] Failed:", data["response"])

if __name__ == "__main__":
    create_address_object(
        name="WEB-SERVER-01",
        ip="10.10.10.100/32",
        description="Web server in DMZ - created via API"
    )
Lab 5

Commit Configuration

POSTtype=commit

Push candidate config to running config. Equivalent of clicking "Commit" in the GUI. Polls for job completion automatically.

API URL
POST https://{FW_IP}/api/?type=commit&cmd=<commit></commit>&key={API_KEY}
ℹ Commit returns a Job ID immediately. The actual commit runs async — poll until status is FIN.
Script
labs/lab5_commit.py
import requests, xmltodict, urllib3, time
from config import API_KEY, BASE_URL, VERIFY_SSL

urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

def commit_config():
    params = {
        "type": "commit",
        "cmd" : "<commit></commit>",
        "key" : API_KEY,
    }
    resp = requests.post(BASE_URL, params=params, verify=VERIFY_SSL)
    data = xmltodict.parse(resp.text)

    if data["response"]["@status"] != "success":
        print("[-] Commit failed:", data); return

    job_id = data["response"]["result"]["job"]
    print(f"[+] Commit started — Job ID: {job_id}")
    poll_job(job_id)

def poll_job(job_id, interval=5, max_wait=120):
    elapsed = 0
    while elapsed < max_wait:
        params = {
            "type": "op",
            "cmd" : f"<show><jobs><id>{job_id}</id></jobs></show>",
            "key" : API_KEY,
        }
        resp   = requests.get(BASE_URL, params=params, verify=VERIFY_SSL)
        data   = xmltodict.parse(resp.text)
        job    = data["response"]["result"]["job"]
        status = job.get("status", "PEND")
        prog   = job.get("progress", "0")
        print(f"    {status} | {prog}%")
        if status == "FIN":
            print(f"[+] Complete — {job.get('result','OK')}")
            return
        time.sleep(interval)
        elapsed += interval
    print("[-] Timed out.")

if __name__ == "__main__":
    commit_config()
Lab 6

Get System Information

GETtype=op

Pull device-level information including hostname, model, serial number, PAN-OS version, and uptime.

API URL
https://{FW_IP}/api/?type=op&cmd=<show><system><info/></system></show>&key={API_KEY}
Script
labs/lab6_system_info.py
from api_client import api_get

def get_system_info():
    params = {
        "type": "op",
        "cmd" : "<show><system><info/></system></show>",
    }
    data = api_get(params)
    info = data["response"]["result"]["system"]

    fields = [
        ("Hostname",       "hostname"),
        ("Model",          "model"),
        ("Serial Number",  "serial"),
        ("PAN-OS Version", "sw-version"),
        ("IP Address",     "ip-address"),
        ("Uptime",         "uptime"),
        ("Threat Version", "threat-version"),
        ("App Version",    "app-version"),
    ]

    print("\n" + "=" * 40)
    print("  FIREWALL SYSTEM INFORMATION")
    print("=" * 40)
    for label, key in fields:
        print(f"  {label:<18}: {info.get(key,'N/A')}")
    print("=" * 40)

if __name__ == "__main__":
    get_system_info()

3-Step Formula for Any New Lab

Building a New API URL

Find the CLI command

SSH to the firewall and run the CLI command you want to automate. Example: show arp all

Convert CLI → XML tags

Each word becomes an XML tag. show arp all → <show><arp><entry name='all'/></arp></show>

Plug into URL template

Use type=op for read-only commands. Use type=config&action=set for config changes. Pass cmd or xpath+element accordingly.

✦ Pro tip: The PAN-OS API browser at https://{FW_IP}/api shows all available XPaths and lets you test commands interactively before scripting them.
